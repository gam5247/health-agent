# Macro Review Packet: GPT E2E Teacher Labels

## User Goal

Create a complete, testable answer key for a clinical-trial matching
multi-agent system. The answer key should be generated by GPT, not by a
deterministic rule fixture, and should cover 100 synthetic patients at contest
prototype scale.

The expected workflow under test:

- criteria parser agent
- patient information understanding agent
- inference and matching agent
- follow-up question generation agent
- simulated patient interaction agent
- recommendation agent
- result explanation agent

## Produced Artifact

The repository now contains `artifacts/gpt-e2e-teacher-labeling/` with:

- `gpt_e2e_teacher_input_100.json`: the 100-patient input given to GPT.
- `gpt_e2e_teacher_labels_100.jsonl`: final answer key, one patient per line.
- `gpt_e2e_teacher_labels_100.json`: same content as one JSON document.
- `gpt_e2e_teacher_labels_100_summary.json`: validation summary.
- `inputs/`: 20 input batches, 5 patients each.
- `requests/`: batch request files.
- `responses/`: normalized, validated GPT responses for each batch.

Raw ChatGPT response captures are intentionally ignored by git.

## Answer-Key Unit

- 100 synthetic patients

Each patient record contains:

## Code Added

- `scripts/build_gpt_e2e_teacher_requests.py`
  - Builds 100-patient GPT input and 20 batch files from processed synthetic patients and retrieved candidate trials.
- `scripts/ingest_gpt_e2e_teacher_response.py`
  - Extracts ChatGPT answer text from `agbrowse` captures.
  - Normalizes GPT output variants.
  - Validates required patients, candidate trials, criterion IDs, and required agent traces.
- `scripts/combine_gpt_e2e_teacher_labels.py`
  - Combines 20 validated batches into the final 100-patient answer key.
  - Revalidates global coverage and writes summary counts.
- `src/health_agent/teacher_student.py`
  - Adds teacher/student pair-building, deterministic baseline labeling, and optional API client helpers for later student-model experiments.
- `tests/test_teacher_student.py`
  - Adds focused tests for pair construction and deterministic teacher/student comparison.

## Known Constraints

- This is a synthetic prototype-scale answer key, not clinical truth.
- GPT output is used as a teacher/silver-label answer key for workflow testing,
  not as a real medical eligibility decision.
- Structural validation errors are zero. Semantic consistency warnings are
  tracked separately because some GPT final labels use the supplied raw criteria
  excerpt in addition to the normalized `criteria_to_assess` subset.
- Raw ChatGPT captures are not committed because they are large intermediate
  files and duplicate the normalized validated responses.
- The answer key validates structural coverage, not clinical correctness by a
  human oncologist.
